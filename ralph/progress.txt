# Ralph Progress — GeekVault SaaS Redesign Plan
# Branch: ralph/saas-redesign-plan
# Started: 2026-03-21

## Codebase Patterns
- CSS custom properties defined in `src/web/src/index.css` with `:root` (light) and `.dark` scopes
- Sidebar has its own token set (`--sidebar-*`) separate from main theme
- Components in `ds/` are design-system-level; `ui/` holds shadcn/ui primitives
- All pages use react-i18next — every UI string needs keys in both `en.json` and `pt.json`
- Feature pages are in `src/web/src/features/{domain}/` with co-located sub-components
- Animation uses Framer Motion wrappers from `ds/motion.tsx` — respects `prefers-reduced-motion`
- Layout breakpoints: mobile (<768px uses Header+Sheet), tablet/desktop (>=768px uses Sidebar+TopToolbar)
- Sidebar collapse state persisted in localStorage key `"geekvault-sidebar-collapsed"`
- Chart colors in dashboard are hardcoded — not from CSS tokens (dark mode issue)
- Empty states inconsistent: some pages show EmptyState component, dashboard sub-sections return null
- New palette uses Tailwind "stone" scale (warm grays) — all neutrals must be stone, not slate/zinc/gray
- Amber accent: #D97706 (light) / #F59E0B (dark) — always brighter on dark backgrounds
- New semantic tokens: `--warning` and `--info` (not in current codebase, will need adding)
- Display font changing from Nunito → Plus Jakarta Sans (warmer geometric, more premium)
- Body font remains Inter — proven for UI, has tabular figures
- Shadow tokens are NEW — current codebase has zero shadow custom properties
- Dark mode shadows use `rgba(0,0,0,...)` at higher opacity; light mode uses warm-tinted `rgba(28,25,23,...)`
- Border radius scale tightened: --radius-md now 8px (was 10px), --radius-lg now 12px (was 16px)
- `--radius` base variable being removed — all radius tokens are now explicit values
- Spacing scale adds --space-0.5, --space-1.5, --space-16, --space-20, --space-24 (new tokens)
- Motion: 5 duration tokens (instant 50ms, fast 150ms, normal 250ms, slow 400ms, deliberate 600ms)
- Motion: 4 spring configs (default, gentle, bouncy, stiff) — always specify `type: "spring"` in Framer Motion
- Motion: Reduced-motion check is module-level in motion.tsx — all new animation wrappers must use `getVariants()`
- Motion: Skeleton pulse must use custom CSS keyframe, NOT Tailwind `animate-pulse` (no reduced-motion respect)
- Motion: Page transitions use AnimatePresence `mode="wait"` — sequential exit→enter, no crossfade
- cmdk `CommandDialog` wraps Radix Dialog — animate via `forceMount` + AnimatePresence, not direct Framer Motion
- Command palette recent items use `sessionStorage` key `"geekvault-recent-pages"` — resets per session
- Page section spacing: `--space-8` (32px) desktop, `--space-6` (24px) mobile — consistent across all pages
- Chart fills must use `hsl(var(--chart-N))` syntax, never hardcoded hex — automatic dark mode support
- Admin analytics reuses StatCard, DataTable, Badge, chart tooltip from user-facing components — no admin-specific UI primitives
- Segmented button groups (time range selector) use `bg-muted` track with `bg-card` active pill + `--shadow-sm` elevation
- Audit log action badges: CRUD=semantic colors, auth=muted, import/export=accent, settings=warning — consistent classification system
- Inline row expansion (accordion) for detail views in tables — use AnimatePresence with height 0→auto animation
- Server-side pagination for potentially large datasets (audit logs); client-side for small bounded datasets (user management)
- Collection cards with onClick on divs need `tabIndex={0}` + `role="link"` + `onKeyDown` for keyboard access
- design-pages.md is the central file for all page-level designs — append new page sections to it
- Grid cards on collections page use `--radius-xl` (16px) for gallery feel — other pages use `--radius-lg` (12px)
- Gradient fallback rotation: `index % 4` for visual variety when cover images are missing
- Continuous hover effects (image zoom) should use CSS transitions, NOT Framer Motion — smoother for mouse-tracking states
- List view tables with image-heavy data should include small thumbnails (40×40px) in the primary column
- Copy card hover-reveal actions need `sm:opacity-0 sm:group-hover:opacity-100` for mobile touch support (not plain `opacity-0`)
- Breadcrumbs only used on catalog item detail page (3 levels deep) — all other pages use sidebar highlight + back links
- Financial values: use `font-variant-numeric: tabular-nums` on price/value displays (Inter supports natively)
- Lightbox components must use `createPortal` to escape overflow containers and keyboard focus trap
- Inline editing: optimistic update + rollback — no success toast, toast on error only
- Image upload zone: `grid-cols-4 sm:grid-cols-5` for preview grid (not flex-wrap — prevents ragged rows)
- Hover-reveal buttons: add `group-focus-within:opacity-100` alongside `group-hover:opacity-100` for keyboard accessibility
- Currency prefix pattern: `absolute left-3 pointer-events-none` + `pl-7` on input
- Progress ring SVG: `stroke-dasharray` = `{pct/100 * circumference} {circumference}`, rotate -90deg to start at top
- Completion tier thresholds: <50% muted, 50-99% accent, 100% success — used across rings, bars, and text colors
- Dashed border = incomplete/missing slot; solid border = owned/acquired — visual language for sets
- Missing items use `bg-card/50` to visually recede; owned items get `border-l-3 border-l-success` left stripe
- Wishlist quick-add: optimistic "Added" → "In Wishlist" (disabled) — no toast on success, toast on error only
- Wishlist cards get subtle `hover:shadow-md` (no translate lift) — data cards, not navigable destinations (same as copy cards)
- Chevron rotation animation (single icon + Framer Motion rotate) is smoother than swapping ChevronDown/ChevronRight icons
- Notes fields should use Textarea (not Input) for multi-line content — `rows={3} resize-y`
- Priority color dots in Select options: `w-2 h-2 rounded-full` inline with flex layout inside SelectItem
- Date-range client-side filtering: use `item.id` descending as creation-order proxy when `createdAt` is unavailable
- Collection types cards: no lift on hover (settings/management cards, not gallery destinations)
- Custom field type badges use color-coded pills: text=info, number=accent, date=success, enum=chart-5, boolean=muted, image_url=chart-6
- Icon picker for collection types: curated 25-icon subset of Lucide, displayed in 4-column grid popover
- Profile page uses card-per-section layout with 2-column grid (avatar left, form cards right)
- Theme toggle applies immediately via context — does NOT require Save button click
- Auth decorative panel: radial gradient layers on stone-900, NOT flat/linear gradient — creates depth and warmth
- Implementation PRD lives at `tasks/prd-saas-redesign-implementation.md` — 34 stories in 8 phases, all referencing design-*.md specs
- Phase dependency order: tokens → motion → shell → command palette → core pages → secondary pages → admin → polish
- Admin feature directory (`src/web/src/features/admin/`) is NEW — must be created during implementation
- Two new ds/ components needed: `image-gallery-lightbox.tsx` and `set-progress-ring.tsx` — create before pages that use them
- Auth floating shapes: pure CSS `@keyframes float` with 20s duration and staggered `animation-delay` — no JS needed
- Auth layout: decorative panel hidden below lg breakpoint — replaced by compact brand header above form
- Disabled/read-only inputs: `opacity-60` + lock icon positioned absolutely in input wrapper
- Admin uses sidebar context switch pattern: `navGroups` vs `adminNavGroups` based on `pathname.startsWith("/admin")`
- Admin nav swap transition: AnimatePresence + fast fade (100ms) on nav group container — not instant swap
- Admin differentiation uses 3 signals: sidebar badge, toolbar pill badge, mobile header text — no color shift
- Admin routes: `/admin` → redirect to `/admin/users`, silent redirect for non-admins (no toast)
- Admin pages in `src/web/src/features/admin/` with `components/` subfolder
- User detail uses Sheet (side panel) not page navigation — keeps admin in list context
- Settings use card-per-section pattern with independent save per section — same as Profile page
- Unsaved changes tracked via form state diffing — amber dot indicator next to section title
- Admin data tables: 56px row height (taller than dashboard's 48px) to accommodate avatar + 2-line text
- Role badge: Admin = accent (amber), User = secondary (muted) — admin badges pop via contrast
- Status dot (not badge): `w-2 h-2 rounded-full` inline with label — status is secondary to role
- Delete user requires typed name confirmation — prevents accidental destructive action
- Settings page max-width: `max-w-3xl mx-auto` — narrow focused layout for forms
- Toggle switches: label+help left, switch right, `flex items-start justify-between gap-4`
