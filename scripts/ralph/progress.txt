## Codebase Patterns
- Auth uses `UserManager<User>` directly (no SignInManager) with `CheckPasswordAsync` for login
- User entity is in `Entities/Security/User.cs` with namespace `GeekVault.Api.Entities.Security`
- Tests use `WebApplicationFactory<Program>` with in-memory database via `TestFactory<TMarker>`
- Program.cs is a single minimal API file with all endpoints; DTOs are in `DTOs/` folder organized by domain
- DTOs use `public record` with namespaces: `GeekVault.Api.DTOs.Security`, `GeekVault.Api.DTOs.Vault`, `GeekVault.Api.DTOs.Common`
- All Vault entities are in `Entities/Vault/` with namespace `GeekVault.Api.Entities.Vault`
- Source files from previous branch need to be checked out explicitly when creating new feature branches
- Database schemas: Security (Users), Vault (collection tables), EF (MigrationsHistory)
- `ToTable("Name", "Schema")` in OnModelCreating to set schema per entity
- `MigrationsHistoryTable("MigrationsHistory", "EF")` in UseSqlServer options
- No .sln file â€” build/test individual .csproj files directly
- Repositories are in `Repositories/{Domain}/` with interface + implementation pairs, registered as `AddScoped` in Program.cs
- Vault repositories inject `ApplicationDbContext`; Security repositories inject `UserManager<User>`
- Services are in `Services/{Domain}/` with interface + implementation pairs, registered as `AddScoped` in Program.cs
- Services inject repositories (not DbContext or UserManager directly); AuthService also injects IConfiguration for JWT
- CatalogItem entity has NO OwnedCopies navigation property â€” use `repo.Query()` IQueryable for cross-entity query composition
- Services use tuple return patterns: `(Response?, Error?)` or `(Response?, NotFound, Error?)` for operations that can fail
- Controllers are static classes in `Controllers/{Domain}/` with `MapXxxEndpoints(this IEndpointRouteBuilder app)` extension methods â€” inject services, not lower-level deps
