## Codebase Patterns
- CSS tokens use hex colors (not OKLch) in the branded palette — keep consistent when adding new tokens
- SortableList uses `renderItem(item, { dragHandleProps, isDragging })` — spread `dragHandleProps` onto the drag handle element (e.g. a GripVertical icon button)
- Font imports via @fontsource packages in index.css, referenced as CSS custom properties (--font-display, --font-body)
- Spacing tokens use --space-N naming (1-12), radius tokens use --radius-sm/md/lg/full
- Sidebar tokens use dark navy in BOTH light and dark themes (always dark sidebar)
- Sidebar toggle from outside: `window.dispatchEvent(new Event("toggle-sidebar"))` — sidebar listens via useEffect
- Command palette at `src/components/layout/command-palette.tsx` — rendered in app-layout.tsx, opened with Cmd+K / Ctrl+K
- Use `.font-display` utility class for heading font (Nunito), body uses Inter by default via font-family on body
- Success color tokens (--success, --success-foreground) added alongside existing destructive tokens
- DS components use `cva` for variants, `cn` for className merging, `forwardRef` — same patterns as shadcn/ui components
- DS barrel export at `@/components/ds` — add new components to `ds/index.ts`
- Radix UI wrappers use Portal for floating elements (Tooltip, Select, DropdownMenu) — ensures proper z-index stacking
- TooltipProvider must be rendered at app root when using Tooltip components
- Generic forwardRef components use inner function + type assertion cast: `const Comp = React.forwardRef(InnerFn) as <T>(props: Props<T>) => ReactElement`
- Motion components use `Variants` type from framer-motion for variant objects — do not use `Record<string, unknown>`
- `prefersReducedMotion` is checked once at module load via `window.matchMedia` — all motion components use it to skip animations
- Toast notifications: use `import { toast } from "@/components/ds"` then `toast.success()`, `toast.error()`, `toast.info()`, `toast.warning()`
- `staggerItemVariants` is exported for use as variants on child `motion.div` elements inside `StaggerChildren`
- Auth pages use `AuthLayout` wrapper from `@/features/auth/auth-layout` for split-screen brand layout (reuse for login/register)
- Page components live in `src/features/{feature}/{name}-page.tsx`, layout components in `src/components/layout/`
- App.tsx imports pages from `./features/{feature}/{name}-page`; layout from `./components/layout/app-layout`
- TypeScript `erasableSyntaxOnly` is enabled — no parameter properties, no enums; use explicit field declarations
- `fetchApi<T>(endpoint, options)` from `@/lib/api` — auto-injects auth token, sets Content-Type, parses JSON, throws `ApiError` on non-OK
- Hooks barrel at `@/hooks` — `useDebounce`, `useMediaQuery`
- `TooltipProvider` is at app root in main.tsx — no need to add it per-component
- Sidebar exports: `Sidebar` (desktop) and `MobileSidebarContent` (for Sheet in header.tsx)
- Sidebar collapse state stored in localStorage key `geekvault-sidebar-collapsed`
- Feature sub-components go in `src/features/{feature}/components/` — e.g. dashboard decomposed into stats-row, charts-section, etc.
- View mode toggle persisted per-entity to localStorage: `geekvault-view-{id}` — use same pattern for any page needing grid/table toggle
- DS Select requires non-empty string values — use `"all"` as sentinel and handle in `updateParam` to delete URL param when selected
- Accordion pattern: use `AnimatePresence` + `motion.div` with `height: 0/auto` + `opacity: 0/1` for expand/collapse; track expanded IDs in `Set<number>`
- Drag-and-drop file upload: `onDragOver`/`onDragLeave`/`onDrop` on a div + hidden `<input type="file" ref={fileInputRef}>` + `onClick={() => fileInputRef.current?.click()}`
- Multi-step wizard: numeric `step` state + step config array `[{ num, label }]` for rendering step indicator with numbered circles and connector lines
- `ConfirmDialog` from `@/components/ui/confirm-dialog` requires `cancelLabel` prop — use for all delete confirmations instead of building from scratch
- Priority badge mapping: 1=High (destructive), 2=Medium (accent), >=3=Low (default) — use with DS Badge variants
- Reorder endpoints pattern: POST `/api/.../reorder` with `{ itemIds: int[] }`, updates SortOrder = array index, returns 204 NoContent
- New entities with SortOrder: auto-assign `GetMaxSortOrderAsync(parentId) + 1` on create; default sort by SortOrder then Id
