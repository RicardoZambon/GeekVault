## Codebase Patterns
- .NET 8 Web API project lives at `src/api/GeekVault.Api/`
- DbContext is `ApplicationDbContext` in `Data/ApplicationDbContext.cs`
- Connection string key: `DefaultConnection` in appsettings.json
- SQL Server: `localhost,1433`, user `sa`, password `P@ssw0rd`, database `ApplicationDB`
- Uses minimal API style (not controllers)
- Build with `dotnet build` from the project directory
- DbContext inherits from `IdentityDbContext<ApplicationUser>` — always call `base.OnModelCreating()` in OnModelCreating
- Identity registered with `AddIdentity<ApplicationUser, IdentityRole>()` in Program.cs
- Migrations generated with `dotnet ef migrations add <Name>` from the project directory
- Use `OwnsMany` + `ToJson()` for JSON columns storing lists of owned types in EF Core 8
- Entity models live in `Models/` directory under namespace `GeekVault.Api.Models`
- Visibility enum stored as string conversion in EF Core: `.HasConversion<string>()`
- Foreign keys: use `OnDelete(DeleteBehavior.Restrict)` for reference data (CollectionType), `Cascade` for ownership (User)
- Decimal monetary fields: use `HasColumnType("decimal(18,2)")` for price/value columns
- Enums stored as strings: use `.HasConversion<string>().HasMaxLength(20)`
- JWT config in `appsettings.json` under `Jwt` section (Key, Issuer, Audience, ExpiresInMinutes)
- Auth endpoints use minimal API style (not controllers) with `MapPost`/`MapGet` in Program.cs
- Test project at `src/api/GeekVault.Api.Tests/` uses xUnit + WebApplicationFactory + InMemory DB
- Use custom `WebApplicationFactory` subclass (not `IClassFixture<WebApplicationFactory<Program>>` with `WithWebHostBuilder`) to share DB across tests
- `Program` class exposed to tests via `Properties/ProgramPartial.cs` (`public partial class Program { }`)
- Protected endpoints use `.RequireAuthorization()` — returns 401 without valid JWT Bearer token
- Use `TestFactory<TMarker>` generic pattern for per-test-class factory isolation (avoids xUnit parallel test contention)
- Avatar uploads go to `wwwroot/uploads/` — gitignored; use `IWebHostEnvironment` to resolve paths
- Use `.DisableAntiforgery()` on file upload endpoints to avoid CSRF issues in API context
- CRUD endpoints pattern: filter by UserId for ownership, validate input in handler, return DTO records
- Nested resource endpoints: verify parent ownership first, then operate on child resource
- Use `.Include()` on parent entity when needing to access navigation properties for validation
- Each test method uses a unique user email to avoid parallel test conflicts
- List endpoints return `PaginatedResponse<T>` (Items, TotalCount, Page, PageSize) — tests use `PaginatedResult<T>` to deserialize
