## Codebase Patterns
- Backend: .NET 8.0 minimal API with EF Core + SQL Server. Build: `cd src/api/GeekVault.Api && dotnet build`. Test: `cd src/api/GeekVault.Api.Tests && dotnet test`
- Frontend: React 19 + TypeScript + Vite 8 + Tailwind CSS v4. Build: `cd src/web && npx tsc -b`. Test: `cd src/web && npm test`
- i18n: All UI strings go in both `src/web/src/i18n/locales/en.json` and `pt.json`
- Existing patterns to follow: CatalogItemsService.ReorderAsync for reorder logic, SetsService for completion % calculation
- CollectionResponse is a positional record — all constructor call sites are in CollectionsService only
- Use GetByIdAndUserIdWithTypeAsync (not GetByIdAndUserIdAsync) when you need CollectionType.Name in the response
- CollectionWithCounts DTO in ICollectionsRepository is the projected query pattern for GetAll
- OwnedCopies count subquery pattern: `_db.OwnedCopies.Count(oc => _db.CatalogItems.Any(ci => ci.Id == oc.CatalogItemId && ci.CollectionId == c.Id))`
- Owned copies endpoint: POST /api/items/{catalogItemId}/copies (requires Condition field)
- EF Core migrations: use `defaultValueSql: "GETUTCDATE()"` instead of `defaultValue` for UTC now defaults on existing rows
- Catalog items endpoint: POST /api/collections/{collectionId}/items (requires Identifier field)
- Both repositories share the same scoped DbContext — setting UpdatedAt on a Collection loaded by one repo and calling SaveChangesAsync on another repo persists all tracked changes
- Collections reorder endpoint: POST /api/collections/reorder with `{ CollectionIds: [int] }` — default sort is now sortOrder
- SortableList gridClassName doesn't accept inline styles — use Tailwind arbitrary property for custom grid template columns
- i18n t() function accepts interpolation params: `t("key", { name: "value" })` — the cmdk CommandItem value prop is what doesn't support interpolation
- skeleton-pulse class is the project standard for loading skeletons (not Tailwind's animate-pulse)
- Motion tokens (springs, durations, easings) are exported from ds/motion.tsx and re-exported via ds/index.ts
- Frontend collections page: isCustomSort (sortBy === "sortOrder") toggles between SortableList (draggable) and StaggerChildren (animated)
- Framer Motion per-variant transitions: embed `transition` inside each variant state (animate/exit) to get different enter/exit timing — remove top-level `transition` prop
- SortableList renderItem accepts optional 3rd param (index) for positional styling
- Warm gradient fallbacks: 4 gradients cycling by index % 4 using --accent and --chart-N CSS variables
- Design specs are in tasks/design-*.md files — read them for visual details before implementing each story
- DB now has role tables (Roles, UserRoles in Security schema). IdentityDbContext<User, IdentityRole, string> is the base class.
- JWT tokens contain NameIdentifier, Email, Jti, and Role claims. Roles seeded on startup: "Admin", "User"
- New users get "User" role on registration via AddToRoleAsync
- AdminOnly authorization policy: RequireRole("Admin") — use .RequireAuthorization("AdminOnly") on admin endpoints
- GET /api/auth/me returns role field (first role from UserManager.GetRolesAsync)
- .NET 8 runtime must be installed alongside .NET 9 SDK — use dotnet-install.sh to add it. DOTNET_ROLL_FORWARD=Major does NOT work for tests (PipeWriter incompatibility)
- dotnet test with FUSE mount: use -p:OutputPath=/tmp/... to avoid DLL lock from VS Code build host, and -p:Threshold=0 to skip coverage gate when running subset of tests
- @fontsource packages: import specific weight files (e.g., `@fontsource/plus-jakarta-sans/400.css`) rather than default index for tree-shaking
- Full test suite may timeout in dev container — run individual test files to verify
- Wishlist API: POST /api/collections/{collectionId}/wishlist with body { name, catalogItemId, priority (1-3), targetPrice, notes }
- Collection detail page test file is too large to run in the dev container — typecheck (npx tsc -b) is the primary verification method
- ImageGalleryLightbox: portal-rendered, takes images[], initialIndex, open, onClose — use createPortal(content, document.body)
- SetProgressRing: takes percentage, size ("sm"|"lg"), showLabel — exports getCompletionTier() for reuse in set cards/pages
- durations object from motion.tsx contains numbers in seconds (e.g., fast: 0.15), not strings — use directly with Framer Motion transition.duration
- ConfirmDialog supports `children` prop for rendering extra content (warnings, alerts) between description and action buttons
- ICON_MAP in collection-types-page: maps Lucide icon name strings to components — TypeIcon helper renders by name or falls back to emoji/Layers default
- Admin domain: Entities/Admin, Repositories/Admin, Services/Admin, Controllers/Admin, DTOs/Admin — registered via AddAdminServices() extension method
- AuditLog entity in Audit schema: Id, Timestamp, UserId, Action, TargetType, TargetId, Details (JSON string), IpAddress — indexed on UserId, Timestamp, Action
- IAuditLogService.LogActionAsync(userId, action, targetType, targetId?, details?, ipAddress?) — fire-and-forget audit logging
